**使っているモデル**
このコードのクレーター劣化は、かなり素直な **2次元線形拡散モデル**です。地形高度を `u(x, y, t)` とすると、実装上のモデルはほぼこれです。
 
```text
∂u/∂t = κ(D) ∇²u
      = κ(D) (∂²u/∂x² + ∂²u/∂y²)
```
 
つまり、熱方程式型の地形拡散です。崖やリムの急な起伏が時間とともに滑らかになり、クレーターの深さが浅くなる、という扱いです。中心実装は [diffusion.py](<C:/Users/tiger/OneDrive/デスクトップ/ドキュメント/thesis_reproduction 2/synthterrain-main/synthterrain-main/src/python/synthterrain/crater/diffusion.py:107>) です。
 
**初期クレーター形状**
初期地形は `FTmod_Crater` で作られます。Fassett & Thomson 型の軸対称プロファイルを、内側、リム、外側で分けた多項式形状として作っています。
 
- `0 <= r <= 0.98`: クレーター内部の多項式
- `0.98 < r <= 1.02`: フラットなリム
- `1.02 < r <= 1.5`: 外側/ejecta 側の多項式
- 床面は指定した深さより下がらないようにクリップ
 
該当箇所は [profile.py](<C:/Users/tiger/OneDrive/デスクトップ/ドキュメント/thesis_reproduction 2/synthterrain-main/synthterrain-main/src/python/synthterrain/crater/profile.py:117>) です。初期の fresh な `d/D` は Stopar et al. 的な段階値で、例えば直径 400 m 以上なら 0.21、200 m 以上なら 0.17、10 m 未満なら 0.10 という表を使っています: [profile.py](<C:/Users/tiger/OneDrive/デスクトップ/ドキュメント/thesis_reproduction 2/synthterrain-main/synthterrain-main/src/python/synthterrain/crater/profile.py:354>)。
 
**拡散係数 κ**
拡散係数 `κ(D)` はクレーター直径 `D` に依存する経験式です。コードでは以下の区分です: [diffusion.py](<C:/Users/tiger/OneDrive/デスクトップ/ドキュメント/thesis_reproduction 2/synthterrain-main/synthterrain-main/src/python/synthterrain/crater/diffusion.py:73>)
 
```text
D <= 11.2:
    κ = 0.0155 / 1e6
 
11.2 < D < 45:
    κ = 1.55e-3 * D^0.974 / 1e6
 
45 <= D < 125:
    κ = 1.23e-3 * D^0.8386 / 1e6
 
125 <= D:
    κ = 5.2e-3 * D^1.3 / 1e6
```
 
最終単位は `m^2/year` として扱われています。
 
**数値解法**
格子はデフォルト `200 x 200`。クレーター直径を `D`、格子数を `N` とすると、
 
```text
dx = dy = 2D / N
dls = dx^2 / 4
κT = κ(D) * age
nsteps = ceil(κT / dls)
```
 
ここで `dls` はコード内で “diffusion length scale” と呼ばれ、陽解法の `κ Δt` に相当します: [diffusion.py](<C:/Users/tiger/OneDrive/デスクトップ/ドキュメント/thesis_reproduction 2/synthterrain-main/synthterrain-main/src/python/synthterrain/crater/diffusion.py:45>)。
 
更新式はこの形です。
 
```text
uᵏ⁺¹[i,j] = uᵏ[i,j]
          + dls * (
              (uᵏ[i+1,j] - 2uᵏ[i,j] + uᵏ[i-1,j]) / dx²
            + (uᵏ[i,j+1] - 2uᵏ[i,j] + uᵏ[i,j-1]) / dx²
            )
```
 
実装は [diffusion.py](<C:/Users/tiger/OneDrive/デスクトップ/ドキュメント/thesis_reproduction 2/synthterrain-main/synthterrain-main/src/python/synthterrain/crater/diffusion.py:219>)。境界セルは更新していないので、外周は固定境界に近い扱いです。
 
**d/D の計算**
各ステップ後、地形配列 `u` の最大標高と最小標高の差をクレーター直径で割っています。
 
```text
d/D = (max(u) - min(u)) / D
```
 
つまり、断面上の厳密なリム高と底面高ではなく、2D 高度場全体の relief を使った `d/D` です: [diffusion.py](<C:/Users/tiger/OneDrive/デスクトップ/ドキュメント/thesis_reproduction 2/synthterrain-main/synthterrain-main/src/python/synthterrain/crater/diffusion.py:224>)。
 
**大量クレーターの近似処理**
全クレーターごとに拡散計算を走らせると重いので、通常は `diffuse_d_over_D_by_bin()` が使われます: [diffusion.py](<C:/Users/tiger/OneDrive/デスクトップ/ドキュメント/thesis_reproduction 2/synthterrain-main/synthterrain-main/src/python/synthterrain/crater/diffusion.py:243>)。
 
処理はこうです。
 
1. 直径を対数ビンに分ける。
2. 各ビンで、クレーター数が 3 以下なら個別に拡散計算。
3. 4 個以上なら、ビン中央値の直径で代表計算を3本走らせる。
4. 3本は初期 `d/D` を `mean`, `mean - std`, `mean + std` にしたもの。
5. 各クレーターの年齢に対応するステップを取り、その時点の代表 `d/D` から正規分布で個体差を付ける。
 
年齢からステップへの対応は、
 
```text
age_step = floor(age * κ / dls)
```
 
です: [diffusion.py](<C:/Users/tiger/OneDrive/デスクトップ/ドキュメント/thesis_reproduction 2/synthterrain-main/synthterrain-main/src/python/synthterrain/crater/diffusion.py:456>)。
 
**年齢推定側**
CSV/XML 変換などで既存クレーターの `d/D` から年齢を推定する場合は逆向きに処理します。`estimate_age()` は fresh な `d/D` から最大年齢まで拡散を走らせ、得られた `d/D(t)` の時系列に観測 `d/D` を当てはめます: [age.py](<C:/Users/tiger/OneDrive/デスクトップ/ドキュメント/thesis_reproduction 2/synthterrain-main/synthterrain-main/src/python/synthterrain/crater/age.py:54>)。
 
合成時は逆で、まずクレーター年齢を `0` から平衡年齢まで一様乱数で選び、その年齢まで拡散させて `d/D` を得ます: [crater/__init__.py](<C:/Users/tiger/OneDrive/デスクトップ/ドキュメント/thesis_reproduction 2/synthterrain-main/synthterrain-main/src/python/synthterrain/crater/__init__.py:161>)。
 
要するに、この実装は「円形 fresh crater profile → 直径依存の線形地形拡散 → relief から d/D を計算」というモデルです。物理的には、非線形斜面輸送や衝突 gardening を明示するというより、クレーター劣化を熱方程式型の平滑化で近似している、と見るのが一番近いです。
